Everything You Need, Nothing Hidden

✨

AI Command Palette

Ask AI to run commands, explain errors, or control your terminal. Streaming responses, pane addressing with #0 or #watermark, and a floating status bar that tracks progress when the palette is dismissed.

🧠

Live AI Summaries

Each pane gets a live summary of its terminal output, updated automatically by the LLM. See at a glance what every pane is doing.

⚡

GPU Rendering

Ghostty's battle-tested Metal/OpenGL renderer with background images, custom shaders, and 120fps. Zero tearing, efficient redraw.

▦

Multi-Pane Grid

Jagged grid layout where each row can have different column counts. Broadcast input, navigate with keyboard, jump with Cmd+1-9.

📋

Pane Overlaying

Stack panes in layers — run a terminal over a web page, overlay notes on a log viewer. Toggle focus between foreground and background.

🧩

Plugin System

10 built-in plugin types with an extensible registry. Terminal, WebView, Notes, File Browser, Git Status, and more.

🔗

Smart Text Selection

Ctrl+click to select a URL or path in one click, even across wrapped lines. Ctrl+drag to extend across line breaks. No more broken selections.

🏷

Pane Watermarks

Tag each pane with a large faint label — environment names, server tags, or project identifiers. Set via config or dynamically via AI.

🔌

Text Tap API

Unix socket API for external control. Subscribe to output, send input, execute actions via JSON. Scriptable automation built in.

🔔

Claude Code Notifications

Native macOS notifications when Claude Code is waiting for input. Uses the Text Tap socket — no osascript needed. Drop-in hook script included.

📊

Context Usage Tracking

Real-time Claude Code context window gauge with color-coded warnings. Track daily and weekly token usage with persistent history. Auto-compact alerts built in.

👁

Agent Monitor

Real-time activity tracking for AI agents running in your panes. See one-liner summaries like “Reading config.zig” or “Running tests” in a floating status pill — even when the command palette is closed.

🌐

Inline Webview Panes

HTTP/HTTPS links open inside a webview pane in the grid instead of jumping to your browser. Stay in context while reading docs, PRs, or dashboards.

🔗

Server URL Detection

Automatically detects dev-server URLs in terminal output and shows a clickable banner. One click opens the server in an inline webview. Supports custom regex patterns for tunnels and proxies.

📨

Send to Pane

Type keystrokes into any pane from an external process. User taps feedback in a Flutter app → MCP server fires → process_queue appears in the Claude Code pane — under one second, fully automatic. Works with shell scripts, Dart, or any Unix socket client.

⚡

Quick Actions

Save frequently-used commands as persistent pill buttons on any pane. Select text, right-click, name it — one click to re-run. Stored in .trm-actions.toml with hot-reload and session persistence.

Crash-Safe Plugin Isolation

Plugins run as subprocesses. If one crashes, trm keeps running. No reload, no restart, no lost work.

trm (host process)
URL Detector in-process
Attention Icon in-process
Send Text Indicator in-process
Quick Actions in-process
Your Plugin subprocess stdin/stdout JSON
🛡

Isolated Processes

Each subprocess plugin runs in its own process with stdin/stdout JSON pipes. A crash is just a pipe EOF — the host detects it instantly.

🔄

Auto-Restart

Crashed plugins restart with exponential backoff (2s, 4s, 8s). After 60 seconds of stable running, the retry counter resets. Three strikes and the plugin stays down — your terminal stays up.

🌍

Any Language

Subprocess plugins are standalone executables. Write them in Python, Rust, Go, or anything else. Read JSON from stdin, write state to stdout. That's the whole API.

Host → Plugin (stdin)
{"type":"terminal_output","pane":0,"text":"...","hash":"abc"}
{"type":"notification","name":"needsAttention","pane":0}
{"type":"stop"}
Plugin → Host (stdout)
{"type":"ready"}
{"type":"state","overlay":"server_url_banner","panes":{"0":["http://localhost:3000"]}}
{"type":"error","message":"regex failed"}

Quick Start

1

Download

Grab the latest release from GitHub:

Download for macOS

Or build from source:

git clone https://github.com/keyvez/trm.git
cd trm
zig build -Doptimize=ReleaseFast
2

Create a config

# ~/.config/trm/trm.toml
title = "My Project"
rows = 2
cols = 2

[[panes]]
title = "Editor"
cwd = "~/projects/myapp"
watermark = "EDIT"

[[panes]]
title = "Server"
initial_commands = ["npm run dev"]

[[panes]]
title = "Tests"
initial_commands = ["zig build test"]

[[panes]]
type = "webview"
title = "Docs"
url = "http://localhost:3000"
3

Launch

# From your project directory
trm

# Or specify a config file
trm --config path/to/config.toml
